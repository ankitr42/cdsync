The User Interface

 

CD Sync Portable comes with a simple, elegant and clean user interface that closely resembles Windows Explorer. The interface has been designed to be explorer like, so that it is easier for you to interact with the program. The main program window appears as shown in the following screen-shot.

 

 

A.    Menu bar and toolbar: The menus on the menu bar contain all the commands and features that the program provides. The File menu contains commands for working with the files contained within the image. The commands in this menu are activated only when some item(s) is selected in the file browser (B in the screen-shot). The Image menu contains commands for creating a new image, deleting an existing image and viewing properties of an image. The View menu lets you adjust how the file browser displays objects. The Tools menu contains commands to export images, import images, export file list, manage image categories, search, and configure other options for the program. The toolbar contains buttons for quick access to major commands on the menu bar. The address bar below the toolbar displays the address of the current location that you’re browsing in the file browser.

B.    The file browser: The file browser lets you browse the image selected in the image browser (E). The file browser can display files in any of the three views: Icons, List or Details. Out of the three views, the details view is the most informative one. It presents the contents of the image in a table like structure similar to the details view of Windows Explorer. You can configure which columns are displayed in the details view using the Tools > Options menu item. When you select an item in the file browser, its basic properties will be displayed in the lower right section of the main window called the file browser information area (depicted by H) in the screen shot. This facility can be turned off by using the application options. If nothing is selected, this area shows information about the directory that you’re browsing. If the extended properties panel (extended properties panel refers to the components C and D combined) is active, and you select a file in the file browser, then its extended properties (if available) as well as a preview (if the file is previewable and the image source is available) will be shown in the panel. Double clicking an item will activate it. The result of activation depends on the type of object that is being activated. For ex., it a folder is double-clicked, it will be opened in the file browser. On the other hand, it a file is double-clicked, it will be opened with its associated program (if the image source is available).

C.    The preview panel: The preview panel lets you preview audio, video and picture file types all at one place. The preview panel can generate previews for about 30 different audio, video and picture file types. It can be configured to automatically play audio and video files when you select them in the file browser.

D.    The extended properties panel: The properties panel displays extended properties (if available) about the selected item in the file browser. The information is presented in a well categorized manner, with related information grouped under proper headings. To store extended information about files, you must select the proper options while creating the image. For more information on how to create a new image see creating a new image.

E.     The image browser: The image browser lists all the images that you’ve in your collection, in a hierarchical tree like structure. The images are grouped under the categories to which they belong. There are three root nodes in the tree. Categorized Images: This node has several sub-nodes that represent the categories that exist. Each category node may in turn contain several nodes representing the images that belong to the category. Uncategorized Images: This node acts as the parent to all those images nodes that have not yet been classified into a particular category. While creating a new image, you can choose to keep it uncategorized. The last root node, Deleted Categories, contains those images that once belonged to some category, but their categories now been deleted. When you delete a particular category, all the images belonging to it are moved under this node. Also when you import images that were exported from another pc, they may belong to categories that do not exist on your machine. In this case you are given the option to create those categories which importing the images. If you choose not to create new categories, then any such images are also added to this node.

F.     The image picture: This area shows the picture (if any is) associated with the image. You can attach a picture with an image when you create the image. The picture can also be changed or removed through the Image Properties dialog box. This feature can be used as an easy identification for the image; for instance, you can add the cover picture of the source media.

G.    Image information area: This area displays information about the image that you’re currently browsing. The displayed information includes the name, source path, volume label of the source media, image category (in-case the image belongs to a deleted category), and image description. H.    File browser information area: This area displays the details of the item(s) selected in the File Browser. This area shows basic file/folder details like, file type, icon, size etc. If no item is selected in the File Browser, it shows the details of the folder you’re browsing.
