Exporting File List

 

With this feature, you can export a listing of the complete directory structure of an image in XML format. This can then be interpreted by any other interpreter program in any form as you find fit. To export the file list for an image follow, you need to bring up the Export Filelist dialog box. To open the Export Filelist dialog box, follow these steps:

 

1.     While in the main window, right click the node representing the image for which you want to export file list. From the resulting context menu, select Export Filelist menu item.

OR

2.     While in the main window, activate the image for which you want to export file list. Now either press Ctrl + X or select the Tools > Export Filelist option. Doing either one of step 1 or 2, will bring up the Export Filelist dialog box:

 

 

3.     In the Export Filelist dialog box, click the Browse button to select a file to save the file list to.

4.     Once you’ve selected the export file, select the file details that you would like to be exported.

5.     Once done, click the Export button to being the export process.

 

XML Schema of the exported file

The exported XML file contains two elements: the <Root> element and the <DirectoryEntry> element. The <Root> element serves as the root element of the XML document. It may contain a number of <DirectoryEntry> elements. Each <DirectoryEntry> element represents a file system object (file/folder). Each <DirectoryEntry> element may in-turn contain several other <DirectoryEntry> elements depending on whether the file system object that it represents is a directory. The attributes of the <Root> and the <DirectoryEntry> elements are explained below:

 

<Root> element: This is the root element of the xml file. It may contain a number of <DirectoryEntry> elements.

Possible attributes:

ImageName: Contains the name of the image from which this list was exported.

Possible child elements:

<DirectoryEntry>

 

<DirectoryEntry> element: This element represents a file system object.

Possible attributes:

Name: Contains the name of the file system object that this <DirectoryEntry> element represents.

Attrubutes: Contains a string representation of the attributes of the file system object. (D = Directory, A = Archive, R = Read Only, S = System, H = Hidden, C = Compressed, E = Encrypted.)

CreatedDate: Contains the creation date of the file system object.

Modified: The modification date of the file system object.

AccessedDate: The accessed date of the file system object.

Artist: The artist information of the file system object.

Album: The album information of the file system object.

AlbumYear: The album year of the file system object.

TrackNumber: The track number of the file system object.

Duration: The duration of the file system object.

Genre: The genre of the file system object.

Bitrate: The bit rate of the file system object.

SampleRate: The sample rate of the file system object.

Dimensions: The dimensions of the file system object.

Author: The author of the file system object.

Title: The title of the file system object.

Subject: The subject of the file system object.

Category: The category of the file system object.

Pages: The number of pages in the file system object.

Comments: The comments on the file system object.

Copyright: The copyright notice of the file system object.

Company: The company of the file system object.

Description: The description of the file system object.

FileVersion: The file version of the file system object.

ProductName: The product name of the file system object.

ProductVersion: The product version of the file system object.

Possible child elements:

<DirectoryEntry>
